# OpenSpec — Project Specification Framework

## Purpose

OpenSpec provides structured project context so AI assistants can deliver accurate, project-aware responses. Every workspace should maintain a project specification.

## How to Use

### Create Project Spec
When working on a new project, define key project metadata:
- Project name, platform, language, frameworks
- Module/package structure
- Architecture patterns used
- Key libraries and dependencies
- Naming conventions

### Keep It Updated
When the project structure changes significantly, update the spec.

## Benefits
- AI responses match your project's actual patterns and conventions
- No need to repeatedly explain project structure
- Consistent code generation that fits your architecture
- Better refactoring suggestions that understand module boundaries
